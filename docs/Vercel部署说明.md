# Vercel 部署说明

本文档用于把 `vercel-mini-games` 项目部署到 `Vercel`。

## 1. 部署前确认

在本地先确认以下几点：

- 已执行 `npm install`
- 已执行 `npm run build`
- 代码已经推送到 GitHub
- 当前项目不依赖数据库和环境变量

## 2. 在 Vercel 导入项目

1. 打开 `Vercel` 控制台
2. 点击 `Add New` -> `Project`
3. 选择 GitHub 中的 `vercel-mini-games` 仓库
4. 点击 `Import`

## 3. 推荐构建设置

这个项目默认就能被识别成 `Next.js`，通常不需要手改配置。

建议检查以下字段：

- `Framework Preset`：`Next.js`
- `Build Command`：留空或使用默认值
- `Output Directory`：留空
- `Install Command`：留空或使用默认值

## 4. 环境变量

当前版本不需要额外环境变量。

如果后续增加：

- 在线排行榜
- 账号系统
- 分析统计
- 第三方接口

再单独补环境变量即可。

## 5. 首次部署

点击 `Deploy` 后，正常情况下会直接开始构建。

如果部署成功，你会拿到一个默认域名，例如：

```text
https://vercel-mini-games-xxx.vercel.app
```

## 6. 部署后检查清单

首次上线后，建议至少检查：

- 首页是否能正常打开
- 5 个游戏路由是否都能正常进入
- `2048` 是否能用键盘操作
- `贪吃蛇` 是否能正常前进、吃食物、失败后重开
- `扫雷` 是否能正常翻格、插旗、切换难度
- `俄罗斯方块` 是否能下落、旋转、消行
- `蜘蛛纸牌` 是否能移动顺子、补牌、自动收走完整顺子

## 7. 自定义域名

如果你有自己的域名，可以在项目设置里添加：

1. 打开项目
2. 进入 `Settings` -> `Domains`
3. 添加你的域名
4. 按 Vercel 提示配置 DNS

## 8. 后续更新发布

后续只要：

1. 本地提交代码
2. 推送到 GitHub 主分支
3. 等待 Vercel 自动重新部署

如果需要手动重发：

- 打开 `Deployments`
- 找到最新一次部署
- 点击 `Redeploy`

## 9. 常见问题

### 9.1 构建失败

先在本地执行：

```bash
npm run build
```

如果本地都过不了，优先修本地错误再重新推送。

### 9.2 页面能打开但交互异常

优先检查：

- 浏览器控制台是否报错
- 是否有 `localStorage` 读写异常
- 是否是移动端手势未覆盖导致的交互差异

### 9.3 想用 Vercel CLI 部署

如果你后续想改用 CLI，可以在本地执行：

```bash
npm i -g vercel
vercel
```

首次会要求登录和绑定项目。

## 10. 当前结论

当前项目已经满足：

- 可本地构建
- 可推送 GitHub
- 可直接导入 Vercel
- 无需环境变量即可首版上线

也就是说，现在已经处于“可以部署”的状态。
